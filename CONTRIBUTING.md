# Contributing to the Project

Thank you for considering contributing to this project! Following the guidelines below will help streamline the contribution process.

## Branch Naming Conventions
- **feature/**: Used for developing new features (e.g., `feature/add-login`).
- **fix/**: Used for bug fixes (e.g., `fix/typo-fix`).
- **docs/**: Used for documentation changes (e.g., `docs/update-readme`).

## Commit Conventions
- Use the present tense (`Add feature` not `Added feature`)
- Use the imperative mood (`Change` not `Changed`)
- Limit the first line to 72 characters or less
- Reference issues and pull requests in the commit message.

Example messages:
- `feature: add user authentication`
- `fix: resolve login bug`

## Pull Request Requirements
- Ensure your branch is up to date with the main branch before submitting.
- Provide a clear description of the changes made and the reason.
- Use `Fixes #issue_number` to automatically close issues when the PR is merged.

## Development Workflow
1. **Fork the repository**: Click the "Fork" button in the top right corner of the repo.
2. **Clone your fork**: Use `git clone <your-fork-url>`.
3. **Create a new branch**: Use `git checkout -b feature/your-feature-name`.
4. **Make your changes**: Commit as per the commit conventions.
5. **Push to your fork**: Use `git push origin feature/your-feature-name`.
6. **Create a pull request**: Go to the original repo and click "New pull request".

Happy coding!
